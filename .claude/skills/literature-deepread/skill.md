# Literature Deep Reading（文献精读流程）

这个 Skill 用于把一篇论文做成可复用、可审阅、可发布的“精读分析”。

## 核心目标

1. 快速判断论文到底在解决什么“真问题”。
2. 分清哪些证据是结论骨架，哪些只是展示工作量或视觉冲击。
3. 给出批判性判断，避免“正确但空泛”的总结。
4. 产出符合博客规范的 Markdown 内容。

## 输出结构（默认）

按以下顺序输出：

1. `## 文献精读：逻辑结构四段法与证据分层`
2. `### 1. 逻辑拆解（每项一句话）`
3. `### 2. 实验证据的“成分分析”`
4. `### 3. 批判性思考`
5. `### 4. 一句话定性核心创新`
6. `### 5. 还能补什么（提精度）`

## 方法框架

### A. 逻辑四段法（每项仅一句）

- **小背景**：研究切入的具体子领域是什么。
- **真问题**：作者指出的核心矛盾是什么，是否成立。
- **课题设计**：作者提出了什么假设或策略。
- **验证方法**：如何通过实验/模拟/对照来验证该假设。

### B. 证据分层法（建议表格化）

把证据分为四层：

- **核心实验（骨架）**：没有它，结论就站不住。
- **求新炫技（首饰）**：增强可视化或新颖性，但不是逻辑必需。
- **交叉印证（肌肉）**：用多模态证据闭环，提高可信度。
- **展示工作量（体力活）**：参数扫描、配方扩展、重复验证等。

### C. 批判性思考（找茬）

至少回答：

1. 性能或结论是否接近/超出理论边界，是否解释充分。
2. 不同证据之间是否存在矛盾或难以自洽之处。
3. 结论是否过度外推，适用范围是否被清楚限定。

### D. 一句话创新定性

禁止“通过XX实现XX提升”式空话；必须点出“打破了什么旧约束/提出了什么新机制/建立了什么可迁移范式”。

## 写作与格式约束

- 优先用简练句，不堆砌术语。
- 证据分层优先使用 Markdown 表格呈现。
- 保持中英文混写可读性；术语首次出现可加英文括注。
- 对关键判断使用适度加粗，但不要整句滥加粗。
- 化学与单位格式遵循项目规范（如 `$\mathrm{Ca^{2+}}$`、`$\mathrm{kg\cdot m^{-3}}$`）。
