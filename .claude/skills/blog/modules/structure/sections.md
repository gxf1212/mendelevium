# 文章章节结构

标准科研博客文章的所有章节，按顺序排列。

## 必须包含的章节

### 1. 标题（一级标题）

```markdown
# [引人入胜但专业的中文标题]
```

**要求**：
- 引人入胜但不标题党
- 一语道破核心卖点
- 信息丰富、包括所有关键点，但别太长
- 少用冒号，就正常人说的一两句话
- 不要带奇怪的引号名词
- 不好翻译的学术词汇就不翻译，如Ramachandran plot、artifact等
- 坚决杜绝：重塑、颠覆、革命、突破、新范式等听着像AI写的词汇
- 全文语气以客观为主，不要夸大promising或贬低

---

### 2. 本文信息

```markdown
## 本文信息
- **标题**: [论文标题中文翻译]
- **作者**: [主要作者]
- 发表时间: [论文发表时间，某年某月某日]
- **单位**: [作者单位，必须标注国家]
- **引用格式**: [完整的学术引用格式]
- **源代码**: [GitHub链接等，如有]
- **Web Server**: [在线工具链接，如有]
```

**所有项都不能少！**

特别注意：
- 源代码链接：如果论文有GitHub等，必须列出
- Web Server：如果有在线工具，必须列出

---

### 3. 摘要

```markdown
## 摘要
> [专业级中文翻译，保持学术严谨性]
> [少量加粗强调核心概念]
```

**要求**：
- 完整翻译论文摘要
- 保持学术严谨性
- 适当加粗关键概念（不超过3-5处）

---

### 4. 核心结论

```markdown
### 核心结论

用清晰的无序列表总结本文的核心结论（3-5条）：
- **结论1**：xxx
- **结论2**：xxx
- **结论3**：xxx
```

**要求**：
- 3-5条核心结论
- 每条加粗关键词
- 简洁明了，一针见血

---

### 5. 背景

```markdown
## 背景

[2-3段深入阐述]

第1段：研究领域的大背景，为什么这个问题重要
第2段：当前存在的技术挑战、理论瓶颈或未解决的关键问题
第3段（可选）：前人工作的局限性
```

**要求**：
- 2-3段，每段100-200字
- 系统性梳理背景
- 说明问题的重要性

---

### 6. 关键科学问题

```markdown
### 关键科学问题

[详细阐述本文旨在解决的核心科学问题]
[不仅罗列，要解释为什么是难点和焦点]
```

**要求**：
- 明确问题是什么
- 说明为什么是难点
- 为什么现在需要解决

---

### 7. 创新点

```markdown
### 创新点

用列表清晰总结本文的主要创新：
- **理论创新**：xxx
- **方法创新**：xxx
- **应用创新**：xxx
```

**要求**：
- 按类型分类（理论/方法/应用）
- 每条简洁明了
- 突出与前人工作的区别

---

### 8. 研究内容（核心部分）

```markdown
## 研究内容

### 核心方法：[方法名称]详解

[详细描述关键方法]
[必要时使用Mermaid流程图]

**数据集/建模过程**：
- xxx
- xxx

**关键步骤**：
1. xxx
2. xxx

### 主要结果

#### [子主题1]

![fig1](文章文件夹/fig1.png)

**图1：[图标题中文翻译]**
- **图1A**：xxx
- **图1B**：xxx
- **颜色说明**：蓝色代表xxx，红色代表xxx

[讲解段落：解读图1的关键发现]

#### [子主题2]

...（重复结构）

### 讨论（可选）

[对研究发现及其意义的深刻阐述]
```

**要求**：
- 方法详述必不可少
- 正文4-8张图，每图都有讲解段落
- 图注必须完整（颜色+子图说明）
- 避免大段话，多用列表
- 图文融合充分

---

### 9. 关键结论与批判性总结

```markdown
## 关键结论与批判性总结

### 主要影响
- **学术影响**：xxx
- **工业影响**：xxx

### 局限性
- xxx
- xxx

### 未来方向
- xxx
- xxx
```

**要求**：
- 客观评价影响
- 指出局限性
- 提出未来方向

---

## 章节顺序

**严格按以下顺序**：
1. 标题
2. 本文信息
3. 摘要
4. 核心结论
5. 背景
6. 关键科学问题
7. 创新点
8. 研究内容
9. 关键结论与批判性总结

## 长度控制

- **主文档**：严格250-300行
- **附录**（如需要）：不限长度

## 附录拆分规则

如果主文档超过300行，拆分以下内容到附录：
1. 技术细节和补充数据
2. SI（补充信息）的内容
3. 详细的方法学对比
4. 完整的图表（正文保留核心图）
5. 公式推导

**不拆分到附录**：
- 核心方法的基本描述
- 主要结果和讨论
- 深入的技术讨论（原Q&A内容应融合到相应章节）

## 内容融合原则

**重要**：不再使用独立的Q&A章节，而是将深入的技术讨论融合到正文相应章节中：

- **方法细节讨论** → 融入"研究内容"的方法部分
- **结果深入解读** → 融入"研究内容"的结果部分
- **潜在争议辨析** → 融入"关键结论与批判性总结"的局限性部分
- **技术细节问题** → 在相关图表讲解后以"需要注意的是..."或"更深层次的考虑是..."的形式补充

**详细融合策略和示例**：参见 `qa_integration.md` 文档。

**简单示例**：
```markdown
### 三、模型性能评估

#### 总能量预测精度

![fig2](fe_nn_figs/fig2.png)

**图2：四种神经网络模型在总能量预测上的性能对比**
- **横轴**：参考能量（DFT计算值）
- **纵轴**：预测能量（神经网络预测值）

从图中可以清楚看出，SchNet+SEE（紫色）的预测点紧密聚集在对角线周围，MAE仅0.037 eV。

**需要注意的是**，这种高精度是在训练分布内取得的。对于训练集之外的配体类型（如PPh₃、bpy），预测误差会略有增加至0.05 eV左右。这提示我们：虽然缩放电子嵌入具有良好的泛化能力，但在应用于完全新的化学空间时，仍需谨慎验证并考虑用少量高精度计算进行微调。

**更深层次的考虑是**，缩放因子α与配体极化率呈正相关这一发现，不仅提供了物理可解释性，也暗示了该方法的适用边界——对于极化率异常低或高的配体体系，可能需要重新训练α值的预测范围。
```
