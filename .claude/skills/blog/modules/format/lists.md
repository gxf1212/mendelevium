# 列表使用规范

## 核心原则

列表不要滥用，是为了层级更清晰，列表太多反而影响易读性。每个item不要太短，而应该是一段话辅以适当加粗更合适。为了列举数据、参数之类的，表格更清晰。但也不能完全没列表，要符合"一条item大概几十个字，一般一段话超过200个汉字才拆成列表或多段"的要求。

## 何时使用列表

### 适合使用列表的场景

**逻辑并列关系明确**：当需要列举多个平行的要点、步骤或类别时，使用列表可以清晰地展示层级关系。例如，列举研究的三大创新点、四个实验步骤等。

**强调结构性信息**：当信息具有明确的分类或层次结构时（如"优势"下分为"高效性"、"准确性"、"全面性"），列表能帮助读者快速把握框架。

**避免过长段落**：当一个段落超过200个汉字且包含3个以上独立要点时，拆分成列表可以提高可读性。

### 不适合使用列表的场景

**单个item太短**：如果每个列表项只有10-20个字（如"温度：300 K"、"压力：1 atm"），应该整合成段落或使用表格。

**列表嵌套过深**：避免三层以上的列表嵌套（如"1. → a. → i."），这会让读者迷失。深层嵌套的内容应该重构为子标题+段落。

**描述性内容**：对于叙述性、解释性的内容（如机制解释、原理阐述），段落形式更自然流畅。

## 列表与段落的转换原则

### 从列表改为段落

**场景1：短列表项**
```markdown
❌ 不推荐
- 温度：300 K
- 压力：1 atm
- 时间步长：2 fs
- 模拟时长：100 ns

✅ 推荐
生产模拟阶段采用Langevin恒温器维持300 K，Berendsen控压器维持1 atm，时间步长2 fs，总模拟时长100 ns。
```

**场景2：嵌套列表**
```markdown
❌ 不推荐
- 优势：
  - 高效：计算成本低
  - 准确：包含统计涨落
  - 全面：探索多条路径

✅ 推荐
弦方法具有三大优势。**高效性**：只需计算平均力，成本不会随维度指数增长。**准确性**：考虑了蛋白质环境的灵活性和统计涨落。**全面性**：能探索多条可能的反应路径，找到能垒最低的那条。
```

### 从段落改为列表

**场景：超过200字且包含明确分点**
```markdown
❌ 不推荐（过长段落）
本研究的优势包括多尺度建模，X射线晶体学提供高分辨率静态结构，MD模拟揭示动态构象变化，QM/MM结合量子力学精度和统计力学采样。同时具有热力学严谨性，FEP精确计算互变异构体相对稳定性，自由能曲线定量描述反应能垒。此外还有机制区分能力，系统比较三种可能机制，定量能垒计算排除不可行路径。

✅ 推荐（适度拆分）
本研究采用的多尺度建模策略具有显著优势：
- **多层次结构描述**：X射线晶体学提供高分辨率静态结构，MD模拟揭示动态构象变化，QM/MM结合量子力学精度和统计力学采样
- **热力学严谨性**：FEP精确计算互变异构体相对稳定性，自由能曲线定量描述反应能垒，统计不确定度评估保证结果可靠性
- **机制区分能力**：系统比较三种可能机制，定量能垒计算排除不可行路径，过渡态几何分析验证化学合理性
```

## 列表项的长度规范

### 最佳长度：50-100个汉字

每个列表项应该是一个完整的句子或短段落，包含主题+说明+细节。

**示例（良好）**：
```markdown
- **初始路径构建**：使用25个节点表示反应路径，通过线性插值在反应物和产物之间构建初始反应坐标。这个初始路径不需要完美，自适应算法会在后续迭代中自动修正
```

### 过短（<30字）：应整合

**示例（需改进）**：
```markdown
❌ 太短
- 节点数：80个
- 力常数：10 kcal/mol
- 模拟时间：500 ps

✅ 改进
伞形采样使用80个窗口，每个窗口的力常数为10 kcal·mol⁻¹，MD模拟时间为500 ps。
```

### 过长（>150字）：应拆分或改段落

**示例（需改进）**：
```markdown
❌ 太长
- 能量最小化阶段分两步进行，第一步为5000步最速下降加5000步共轭梯度且蛋白质重原子施加50 kcal/mol的约束，第二步为2500步最速下降加2500步共轭梯度进行无约束优化，然后是平衡阶段包括NVT升温从0 K至300 K持续50 ps施加10 kcal/mol的弱约束...

✅ 改进（拆分为子列表或段落）
能量最小化阶段分两步进行：第一步为5000步最速下降加5000步共轭梯度，蛋白质重原子施加50 kcal/mol的约束；第二步为2500步最速下降加2500步共轭梯度，无约束优化。随后进入平衡阶段...
```

## 列表间距规范

### 列表项之间不应有空行

```markdown
❌ 不推荐
- 第一项

- 第二项

- 第三项

✅ 推荐
- 第一项
- 第二项
- 第三项
```

### 列表前后应有空行

```markdown
✅ 正确格式
本研究使用以下方法：

- 方法一：描述内容
- 方法二：描述内容
- 方法三：描述内容

实验结果表明...
```

## 列表与表格的选择

### 使用表格的场景

**参数列举**：当需要列举多组参数且每组参数有固定的属性字段时，表格更清晰。

```markdown
✅ 使用表格
| 参数 | 反应物 | 过渡态 | 产物 |
|------|--------|--------|------|
| d(N1-C1) / Å | 3.38 | 2.34 | 1.45 |
| d(C1-O1) / Å | 1.43 | 2.04 | 3.50 |
| 能量 / kcal·mol⁻¹ | 0.0 | 16.7 | -12.3 |
```

**对比数据**：当需要对比多个方法或条件的结果时。

```markdown
✅ 使用表格
| 机制 | 能垒 | 中间体 | 可行性 |
|------|------|--------|--------|
| 机制I | 16.7 | 无 | ✓ 最优 |
| 机制II | 21.5 | 五配位磷 | ✗ 偏高 |
| 机制III | 30.2 | 碳正离子 | ✗ 过高 |
```

### 使用列表的场景

**概念解释**：当每个项目需要较长的说明文字时。

```markdown
✅ 使用列表
弦方法具有四大优势：
- **高效性**：只需计算平均力，即使反应坐标（集合变量）很多也能处理，成本不会随维度指数增长
- **准确性**：考虑了蛋白质环境的灵活性和统计涨落，而不是简单的真空中优化
```

## 实战检查清单

完稿后检查列表使用是否合理：

1. **长度检查**：每个列表项是否在50-100字范围内？过短的是否可以合并为段落或表格？
2. **嵌套检查**：是否有三层以上的嵌套列表？是否可以用子标题+段落替代？
3. **密度检查**：一个章节中列表占比是否超过50%？是否影响阅读流畅性？
4. **间距检查**：列表项之间是否有多余空行？列表前后是否有空行？
5. **语义检查**：列表中的内容是否真的是并列关系？还是逻辑递进或因果关系（应改为段落）？

## 典型错误示例

### 错误1：滥用列表导致碎片化

```markdown
❌ 不推荐
QM/MM方法学：
- QM计算：B3LYP/6-31G*
- MM计算：AMBER ff14SB
- 耦合方式：电子嵌入
- 软件：Gaussian 09

✅ 推荐
QM计算采用B3LYP/6-31G*密度泛函理论方法，MM计算使用AMBER ff14SB力场，耦合方式为电子嵌入（Electrostatic Embedding），软件包为Gaussian 09结合AMBER 16。
```

### 错误2：列表项长度不一致

```markdown
❌ 不推荐
- 第一点：这是一个非常详细的说明，包含了很多背景信息、技术细节和实验参数，总共有超过一百个字的长度，读起来像一个完整的段落
- 第二点：简短说明
- 第三点：稍长一点

✅ 推荐（调整为统一长度）
各列表项保持50-100字的均衡长度，避免有的过长有的过短。
```

### 错误3：本应使用表格的数据

```markdown
❌ 不推荐
- 温度：300 K
- 压力：1 atm
- 时间步长：2 fs
- 截断半径：10 Å
- 静电方法：PME

✅ 推荐（表格）
| 参数 | 数值 |
|------|------|
| 温度 | 300 K |
| 压力 | 1 atm |
| 时间步长 | 2 fs |
| 截断半径 | 10 Å |
| 静电方法 | PME |

或✅ 推荐（段落）
模拟参数设置为：温度300 K，压力1 atm，时间步长2 fs，静电截断半径10 Å，采用PME方法处理长程静电相互作用。
```

## 总结

- **核心原则**：列表是为了清晰层级，不是为了分行
- **长度标准**：每条50-100字，低于30字合并，超过150字拆分
- **使用判断**：超过200字的段落且有3+独立要点 → 列表；否则 → 段落
- **表格优先**：纯数据、参数对比 → 表格；概念解释 → 列表
- **间距规范**：列表内部无空行，列表前后有空行
