# 手动质量检查清单

此清单补充自动化检查，需要手动逐项验证。

## 使用说明

1. 先运行`tools/check_blog_quality.py`
2. 修复所有自动检测到的错误
3. 使用本清单逐项手动检查
4. 所有项目打✓后才算通过

---

## 结构检查

### 基本结构
- [ ] 主文档**严格**250-300行（不是大约，是严格！）
- [ ] 无重复图表编号（Figure 1, Figure 2...按顺序）
- [ ] 有完整的frontmatter

### 本文信息完整性
- [ ] 标题（中英文都有）
- [ ] 作者（主要作者）
- [ ] 发表时间（论文发表日期）
- [ ] 单位（包含国家）
- [ ] 引用格式（完整学术引用）
- [ ] 源代码链接（如有，必须列出）
- [ ] Web Server链接（如有，必须列出）

### 图片和图注
- [ ] **每一张**Figure都有完整图注
- [ ] **每一个**图注包含图标题
- [ ] **每一个**图注包含子图说明（A/B/C）
- [ ] **每一个**图注包含颜色说明
- [ ] **每一个**图片路径正确（相对路径）
- [ ] 正文有4-8张图（不多不少）

---

## 格式检查

### 标点符号
- [ ] **所有**中文内容使用中文标点
- [ ] **所有**引号正确（201C/201D Unicode）
- [ ] 无`Q\&A`（应为`Q&A`）
- [ ] 无`1\.`（应为`1.`）
- [ ] frontmatter和mermaid使用英文标点（这是对的）

### 加粗格式
- [ ] **所有**加粗不包含句号（`**重要**. ✅`）
- [ ] **所有**加粗不包含百分号（`**24**% ✅`）
- [ ] **所有**加粗不包含引号（`"**黑箱**" ✅`）
- [ ] **所有**加粗不包含括号（`**优化**（Optimization） ✅`）
- [ ] 加粗使用适度（不要大段加粗）

### 列表格式
- [ ] **所有**列表无多余空行
- [ ] 列表项之间紧凑排列
- [ ] 嵌套列表缩进一致

### 符号规范
- [ ] 无`~700`（应为"约700个"）
- [ ] 无`52000+`（应为"52000余个"）
- [ ] 无`+30`（应为"30"）
- [ ] 无`z < -3`（应为"z值小于-3"）
- [ ] 无`~+25-30`等混合符号

---

## 公式检查（高危）

### 🚨 最严重错误

- [ ] ❌ **绝对没有**双backslash: `$\\Delta$`
- [ ] ✅ **全部使用**单backslash: `$\Delta$`

### 公式规范

- [ ] 微分符号全部用`$\mathrm{d}\xi$`（不是`$d\xi$`）
- [ ] 化学式全部用`$\ce{NaCl}$`（不是`$NaCl$`）
- [ ] 单位全部用`\mathrm{kcal/mol}`或写在公式外
- [ ] 重要公式有"公式的通俗解释"

### 如何检查双backslash

在编辑器中搜索：`\\\\`（连续4个反斜杠）

如果在文件中看到`\\`（连续2个），就是错误！

### 基本规范

- [ ] 子图中有箭头的用了`direction LR`
- [ ] **绝对没有**`---`分隔符

- [ ] 列表格式正确（`2.随机森林`无空格）

### mindmap特别检查

## 内容质量检查

### 核心内容
- [ ] 核心结论清晰（3-5条列表）
- [ ] 背景阐述充分（2-3段，每段100-200字）
- [ ] 关键科学问题明确
- [ ] 创新点总结清晰

### 图文融合
- [ ] **每张图都有讲解段落**（不只是图注）
- [ ] 图片按顺序出现（不跳跃）
- [ ] 讲解段落50-100字
- [ ] 讲解段落不简单重复图注

### Q&A部分
- [ ] 3-5个深入问题
- [ ] 回答有价值，不重复正文
- [ ] 问题涉及技术细节或潜在争议
- [ ] 回答简洁明了

### 批判性总结
- [ ] 包含影响（学术/工业）
- [ ] 包含局限性（至少2条）
- [ ] 包含未来方向（至少2条）
- [ ] 语气客观，不夸大

### 语言风格
- [ ] 无标题党词汇（重塑、颠覆、革命、突破、新范式）
- [ ] 语气客观专业
- [ ] 术语翻译准确
- [ ] 不过度夸大效果（如"promising"）

## 特别注意事项

### 最容易遗漏的检查

1. **frontmatter的date**：经常忘记改成今天的日期
2. **image和thumbnail一致性**：经常只改一个
3. **每张图的讲解段落**：经常只写图注，忘记写讲解
4. **符号规范化**：`~700`, `+30`等容易遗漏
5. **公式双backslash**：最严重但最容易忽视

### 检查顺序建议

1. **先检查高危错误**（公式双backslash、Mermaid `---`）
2. **再检查结构完整性**（章节、图注）
3. **然后检查格式规范**（标点、加粗、列表）
4. **最后检查内容质量**（图文融合、Q&A、批判性总结）
